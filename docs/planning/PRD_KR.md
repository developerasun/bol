# bol PRD

## 1. 개요 (Executive Summary)
"bol"은 '트릭컬: 시비고(Trickcal: Chibi Go)' 플레이어를 위해 설계된 터미널 기반 가챠 확률 계산기입니다. 이 도구는 게임 내 특유의 가챠 구조와 천장 시스템을 반영하여, 유저가 원하는 캐릭터나 아이템을 획득할 확률을 정확하고 간단하게 추산할 수 있도록 돕는 것을 목표로 합니다.

## 2. 문제 정의 및 목표 (Problem Statement & Goals)
### 문제 (Problem)
트릭컬과 같은 가챠 게임은 복잡한 확률 구조와 천장 보정 로직을 포함하고 있어, 유저가 자신의 실질적인 당첨 확률을 직관적으로 이해하기 어렵습니다. 플레이어들은 종종 현재 보유한 재화로 성공할 가능성이 얼마나 되는지, 혹은 안정적인 획득을 위해 얼마나 더 모아야 하는지 고민하게 됩니다.

### 목표 (Goals)
- 주어진 뽑기 횟수에 따른 성공 확률을 계산해주는 CLI 도구 제공.
- '트릭컬: 시비고' 전용 가챠 확률 및 천장 로직 구현.
- 터미널 환경을 선호하는 파워 유저들을 위한 가볍고 빠른 유틸리티 개발.

## 3. 사용자 스토리 (User Stories)
- **뽑기 확률 확인:** 유저로서 나는 특정 횟수의 뽑기를 진행했을 때 3성 캐릭터를 한 번이라도 획득할 확률이 얼마인지 알고 싶다.
- **재화 계획 수립:** 유저로서 나는 90% 이상의 안정적인 확률로 성공하기 위해 몇 번의 뽑기가 필요한지 확인하고 싶다.
- **천장 반영:** 유저로서 나는 현재까지 쌓인 스택(천장 카운트)을 입력하여, 다음 뽑기에서의 실질적인 확률 향상을 확인하고 싶다.
- **배너 비교:** 유저로서 나는 서로 다른 가챠 배너의 효율성을 비교하고 싶다.

## 4. 기능적 요구사항 (Functional Requirements)
| ID | 요구사항 | 설명 |
|---|---|---|
| FR-1 | 확률 설정 | 타겟 캐릭터의 기본 드랍률(예: 픽업 0.75%)을 설정할 수 있어야 함. |
| FR-2 | 뽑기 입력 | 뽑기 횟수 또는 보유 중인 재화량을 입력받을 수 있어야 함. |
| FR-3 | 천장 로직 | 트릭컬 고유의 천장 시스템(예: X회 이후 확정 지급 등)을 계산에 반영함. |
| FR-4 | 통계 출력 | 1회, 10회, 50회, 100회 및 사용자 지정 횟수별 누적 성공 확률을 표시함. |
| FR-5 | CLI 인터페이스 | 플래그를 통해 신속하게 계산 결과를 얻을 수 있는 깔끔한 인터페이스 제공. |

## 5. 기술적 제약 및 성공 지표 (Technical Constraints & Success Metrics)
### 기술적 제약 (Technical Constraints)
- **언어:** 성능과 안정성을 위해 Rust 언어로 작성.
- **환경:** Linux, macOS, Windows에서 독립 실행 가능한 바이너리로 동작해야 함.
- **의존성:** 작은 바이너리 크기와 빠른 컴파일을 위해 외부 크레이트 사용을 최소화함.

### 성공 지표 (Success Metrics)
- **정확도:** 계산 결과가 게임 내 공시된 가챠 시스템의 수학적 기댓값과 일치해야 함.
- **사용성:** 사용자가 한 줄의 명령어로 결과를 얻을 수 있어야 함 (예: `bol --pulls 100`).
- **성능:** 계산 및 결과 출력이 즉각적으로 이루어져야 함.
